1. Difference between var, let, and const

Answer:
- var: Function-scoped, allows redeclaration, hoisted with undefined.
- let: Block-scoped, cannot be redeclared in the same scope, hoisted but in TDZ.
- const: Block-scoped, must be initialized at declaration, cannot be reassigned.

3. Difference between == and ===

Answer:
- ==: Loose equality, performs type coercion.
- ===: Strict equality, checks both value and type.

______________________________________________________________________________________________________________________________________________
4. Hoisting and TDZ

Answer:
- Hoisting: JavaScript moves declarations to the top of scope. Variables declared with var are hoisted with undefined.
- TDZ (Temporal Dead Zone): Period between entering scope and variable initialization where accessing let/const throws an error.

5. Array Methods

Answer:
- Map vs Filter vs Reduce
- map: Transforms each element, returns new array.
- filter: Returns elements that match condition.
- reduce: Accumulates values into single result.
- Map vs forEach
- map: Returns new array.
- forEach: Executes function but returns undefined.
- Find vs Filter
- find: Returns first matching element.
- filter: Returns all matching elements.
- Some vs Every
- some: Returns true if at least one element matches.
- every: Returns true if all elements match.

8. Difference between Object.freeze() and Object.seal()

Answer:
- Object.freeze(): Prevents adding, removing, or modifying properties.
- Object.seal(): Prevents adding/removing properties, but allows modification of existing ones.

9. Execution Context

Answer:
- JS runs inside an Execution Context (Global, Function).
- Each context has Memory (variable environment) and Code (thread of execution).
- Created in two phases:
- Creation Phase (hoisting, scope setup)
- Execution Phase (code runs line by line)

10. Call Stack

Answer:
- A stack structure that keeps track of function calls.
- When a function is invoked, it’s pushed onto the stack; when finished, it’s popped off.
- Helps manage execution order.

12. Difference between innerText, innerHTML, and textContent

Answer:
- innerText: Returns visible text (respects CSS styles like display:none).
- innerHTML: Returns HTML markup inside element.
- textContent: Returns all text content, regardless of visibility.

13. What are Closures in JavaScript?

Answer:
- A closure is a function that remembers variables from its outer scope even after the outer function has finished executing.
- Example:

--------------------------------------
function outer() {
  let count = 0;
  return function inner() {
    count++;
    return count;
  };
}

const counter = outer();
console.log(counter()); // 1
console.log(counter()); // 2

14. Explain Event Loop in JavaScript

Answer:
- The event loop manages execution of synchronous and asynchronous code.
- Steps:
- Executes synchronous code in the call stack.
- Handles asynchronous tasks (callbacks, promises, timers) from the callback queue or microtask queue.
- Ensures non-blocking behavior.

______________________________________________________________________________________________________________________________________________

15. Difference between null and undefined

Answer:
- undefined: Variable declared but not assigned a value.
- null: Explicitly assigned "no value".
- Example:
------------------------------------
let a;
console.log(a); // undefined
let b = null;
console.log(b); // null
